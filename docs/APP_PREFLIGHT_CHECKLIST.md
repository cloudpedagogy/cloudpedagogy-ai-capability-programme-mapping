# CloudPedagogy Tool – Pre-Flight Checklist

Complete this checklist before releasing any new tool.

---

## A. Structural compliance

- [ ] Uses the 5-part structure (Header, Purpose, Activity, Output, Framing)
- [ ] Tool name follows “AI Capability + [Function]”
- [ ] Purpose block included near the top
- [ ] Framing footer included at the end

---

## B. Language compliance

- [ ] No prescriptive language (“you should”, “the correct approach”)
- [ ] Outputs use conditional phrasing
- [ ] Tone is neutral and reflective
- [ ] No marketing or optimisation language

---

## C. Framework compliance

- [ ] All six AI Capability domains are referenced or available as lenses
- [ ] Domain names match canonical wording exactly
- [ ] Domains are framed as considerations, not scores

---

## D. Output integrity

- [ ] Outputs follow the standard structure
- [ ] Outputs support discussion rather than decision enforcement
- [ ] No implied correctness, ranking, or compliance signals

---

## E. Architectural compliance

- [ ] Runs fully client-side
- [ ] No user accounts required
- [ ] No server-side persistence
- [ ] Local or session storage only (if used)

---

## F. Final check

- [ ] Tool supports judgement, not automation
- [ ] Tool would be appropriate in a professional / governance setting
- [ ] Tool reinforces (not replaces) the AI Capability Framework
